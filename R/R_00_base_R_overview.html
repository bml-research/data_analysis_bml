Unlike the tidyverse, base R needs no extra packages — everything below uses functions that ship with R by default.

📘 Base R Overview

💡 This page provides a structured overview of core base R functions, covering the same operations as the Tidyverse Overview page, but without relying on any external packages. While this overview includes references and examples using base R for completeness and historical context, for modern data science workflows, the use of the Tidyverse suite of packages (such as dplyr, tidyr, and ggplot2) is strongly recommended and advised. While base R is powerful, its syntax can occasionally be inconsistent—such as automatically dropping dimensions when subsetting single columns. The Tidyverse avoids these pitfalls by maintaining predictable data structures (like tibbles) and utilizing a clean, pipe-oriented (%>% or |>) syntax. This approach significantly enhances code readability, makes your scripts easier to maintain, and ensures a more consistent programming experience. Click a category to expand and jump to detailed examples.

The examples shown are performed on the Puromycin dataset. The Puromycin dataset is a classic example dataset built into the base R installation (within the datasets package). It originates from a biochemical experiment and is widely used for demonstrating and testing nonlinear regression models, particularly the Michaelis-Menten kinetics model in enzyme reactions.

📈 Visualization

⬆️ Back to Summary

💡 This section shows how to plot with base R graphics. Click on the triangles to expand.

plot() - The function plot() is used to create visualizations of your data.

Base R has no explicit “layer” system like ggplot2 — instead, a plot is built by calling plot() once to set up the canvas, then adding to it with functions like points(), abline(), legend(), and mtext(), or by passing more arguments to plot() directly. In the example below,the Puromycin dataset is used. Each step below redraws the full plot with one more piece added.

Here is a step-by-step example using R code for a scatter plot with a fitted line:

  • Initialization

Sets up the plotting region and axes, with no data drawn yet.

plot(Puromycin$conc, Puromycin$rate, type = "n", xlab = "", ylab = "")

As you can see, no data are plotted yet. An XY-grid is created and the axes are set.

  • Geometry

Adds the visual representation of the data (points for a scatter plot), using a color per state.

col_map <- c(treated = 1, untreated = 2) # default base R colors

plot(Puromycin$conc, Puromycin$rate,
     col = col_map[Puromycin$state],
     pch = 19, cex = 1.5,
     xlab = "", ylab = "")

As you can see, the data is plotted now. Instead of pch = 19, other values can be used for other point shapes, and lines() or barplot() can be used for other plot types.

  • Statistics

Adds a statistical summary (e.g., a line fit to the data, one per group). For simplicity, we just use a linear model.

col_map <- c(treated = 1, untreated = 2)

plot(Puromycin$conc, Puromycin$rate,
     col = col_map[Puromycin$state],
     pch = 19, cex = 1.5,
     xlab = "", ylab = "")

for (s in names(col_map)) {
  sub <- subset(Puromycin, state == s)
  fit <- lm(rate ~ conc, data = sub)
  abline(fit, col = col_map[s], lwd = 2)
}

Note that for plotting enzyme activity, we normally use the Michaelis-Menten model. This requires more complexity and is omitted in this example. In this case, we used a linear model fit per group with abline(), for demonstration purposes.

  • Scale

Controls the mapping from the data values to the visual properties (e.g., color), made explicit here with a custom color vector and legend().

col_map <- c(treated = "purple", untreated = "darkgreen")

plot(Puromycin$conc, Puromycin$rate,
     col = col_map[Puromycin$state],
     pch = 19, cex = 1.5,
     xlab = "", ylab = "")

for (s in names(col_map)) {
  sub <- subset(Puromycin, state == s)
  fit <- lm(rate ~ conc, data = sub)
  abline(fit, col = col_map[s], lwd = 2)
}

legend("bottomright", legend = names(col_map), col = col_map, pch = 19, title = "Enzyme State")

  • Labels

Adds non-data annotations like titles, axis labels, and a caption, using main/xlab/ylab and mtext().

col_map <- c(treated = "purple", untreated = "darkgreen")

plot(Puromycin$conc, Puromycin$rate,
     col = col_map[Puromycin$state],
     pch = 19, cex = 1.5,
     main = "Reaction Rate vs. Substrate Concentration in Puromycin",
     xlab = "Concentration (ppm)",
     ylab = "Reaction Rate (counts/min)")

for (s in names(col_map)) {
  sub <- subset(Puromycin, state == s)
  fit <- lm(rate ~ conc, data = sub)
  abline(fit, col = col_map[s], lwd = 2)
}

mtext("Effect of enzyme state (treated/untreated)", side = 3, line = 0.5, cex = 0.9)
legend("bottomright", legend = names(col_map), col = col_map, pch = 19, title = "Enzyme State")
mtext("Data from Puromycin dataset", side = 1, line = 4, adj = 1, cex = 0.7)

  • Theme

Controls the non-data graphical elements (e.g., background, grid lines, font size), set globally with par() and grid().

col_map <- c(treated = "purple", untreated = "darkgreen")

par(bg = "white", family = "sans", cex.main = 1.2, cex.lab = 1.05, cex.axis = 0.9)

plot(Puromycin$conc, Puromycin$rate,
     col = col_map[Puromycin$state],
     pch = 19, cex = 1.5,
     main = "Reaction Rate vs. Substrate Concentration in Puromycin",
     xlab = "Concentration (ppm)",
     ylab = "Reaction Rate (counts/min)",
     panel.first = grid(col = "grey90"))

for (s in names(col_map)) {
  sub <- subset(Puromycin, state == s)
  fit <- lm(rate ~ conc, data = sub)
  abline(fit, col = col_map[s], lwd = 2)
}

mtext("Effect of enzyme state (treated/untreated)", side = 3, line = 0.5, cex = 0.9)
legend("bottomright", legend = names(col_map), col = col_map, pch = 19, title = "Enzyme State")
mtext("Data from Puromycin dataset", side = 1, line = 4, adj = 1, cex = 0.7)

This is just a demonstration of the use of the various layers.
